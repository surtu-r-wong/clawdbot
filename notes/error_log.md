## 🚨 重大错误记录

### 1. 数据幻觉错误（2026-02-05）
- **日期**: 2026-02-05
- **查询**: 今天账户盈亏
- **错误类型**: 数据幻觉
- **错误描述**: API返回`浮动盈亏=6050`，但我计算时写成了`60570`（10倍差异）
- **根本原因**: 没有直接使用API数据，而是凭记忆输入数字
- **影响**: 计算结果完全错误
- **修正情况**: 用户指出后修正

### 2. 账号数据错误（2026-02-03）
- **日期**: 2026-02-03
- **查询**: 多au-ag头寸盈亏
- **错误类型**: 数据源错误
- **错误描述**: 把"多au"和"多au-ag"的数据混在一起计算，没有筛选头寸名称
- **根本原因**: 没有按规则筛选头寸名称
- **影响**: 计算结果错误
- **修正情况**: 用户指出后重新计算

### 3. 执行流程错误（多次）
- **日期**: 2026-01-27 至 2026-02-05
- **查询**: 多次查询
- **错误类型**: 执行流程错误
- **错误描述**: 没有读SKILL.md就开始计算，反复犯错
- **根本原因**: 没有遵循"用postgres查询"的触发语机制
- **影响**: 反复犯错，查询效率极低
- **修正情况**: 每次用户都指出，但屡教不改

## 🎯 改进承诺

### 强制规则
1. 每次使用Postgres技能前必须读SKILL.md
2. 查询后立即列出原始数据
3. 计算时每步都引用原始数据
4. 计算完后验证：原始数据 vs 计算使用的数据
5. 发现任何不一致立即警告

### 数据验证规则
1. 必须直接使用API数据，不能凭记忆输入
2. 如果必须重写，必须逐位对比验证
3. 明显异常（如10倍差异）必须立即发现
4. 按账号、头寸名称分别计算，不要合并

### 执行流程规则
1. 使用触发语："用postgres查询：[问题]"
2. 读取 SKILL.md
3. 列出相关公式
4. 按步骤计算
5. 验证数据完整性
6. 输出结果
